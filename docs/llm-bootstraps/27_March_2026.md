## 🧭 Core Architectural Principles

1. **Everything is a CdObj**

   * Files, directories, modules, components
   * Typed via `cd_obj_type`
   * Behavior defined via `cd_obj_role`

2. **Descriptors define structure**

   * ComponentDescriptor unifies controllers, services, etc.
   * CdAppDescriptor includes `directorySignature`

3. **SeedConfig defines subsystem DNA**

   * Drives scanning, generation, transformation
   * Roles define expected structure (controller, service, etc.)

4. **DNA Expression Grammar replaces regex**

   * Expressions resolve roles dynamically
   * Roles act as operators in grammar

5. **ICdRequest is the universal execution contract**

   * CLI → interpreted → ICdRequest
   * AI → direct ICdRequest
   * Execution layer is transport-agnostic

---

## 🔁 Execution Model

Two entry paths:

### CLI Path

User Input → Interpretation Layer → ICdRequest → Execution

### AI Path

AI Output → ICdRequest → Execution

🚨 Rule:
Execution MUST NOT depend on source (CLI, AI, HTTP)

---

## 🧩 ICdRequest Structure (Key Distinction)

* `dat` → Transport payload (raw, external)
* `args` → Execution input (clean, structured)

A resolver layer transforms:

```
dat → args → execution
```
