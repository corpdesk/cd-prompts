## 1. Executive Summary & Abstract

This specification defines the standard data contract and runtime behavioral pattern for managing configuration profiles within the CorpDesk ecosystem. It addresses a critical security and architectural challenge: **preventing the exposure of sensitive credentials (tokens, private keys, API keys) within plain-text configuration files, while maintaining structural flexibility for multi-tenant, language-agnostic components.**

This standard establishes the `ProfileContainer` layout and introduces **CdVault**, a mechanism for deferred, runtime resolution of encrypted or referenced secrets. By abstracting configuration variables into explicit reference paths (`#cdVault['key']`), the CorpDesk framework guarantees that secrets remain isolated within a secure vault domain until explicitly parsed and resolved at the application execution layer.

## 2. Structural Component Specifications (RFC Layout)

### 2.1 The Global Data Contract: `ProfileContainer`

All configuration subsystems within the CorpDesk ecosystem must structure their data storage payloads to conform strictly to the `ProfileContainer` schema. This layout decouples system metadata from specialized application contexts.

```
+-------------------------------------------------------------+
|                      ProfileContainer                       |
+-------------------------------------------------------------+

| - count: number                                             |
+-------------------------------------------------------------+

+-------------------------------------------------------------+
|                        ProfileModel                         |
+-------------------------------------------------------------+
| - cdCliProfileId?: number                                   |
| - cdCliProfileGuid?: string                                 |
| - cdCliProfileName: string                                  |
| - cdCliProfileEnabled: number (0|1)                         |
| - cdCliProfileTypeId: number (CdCliProfileTypes)            |

+-------------------------------------------------------------+
|                        ProfileData                          |
+-------------------------------------------------------------+
```


### 2.3 The Deferred Reference Pattern

Secrets must not be stored nakedly inside the `details` matrix. Instead, fields requiring high security must utilize a string interpolation pointer matching the following regular expression:

```regex
/#cdVault\['(.+?)'\]/

```

**Example Construction:**

```json
{
  "details": {
    "consumerToken": "#cdVault['consumerToken']"
  }
}

```

## 3. Implementation Blueprint & Runtime Lifecycle

### 3.1 Resolution Engine Mechanics

The resolution process scans configuration leaf nodes at execution time. When a tokenized reference string is discovered, the runtime matches it against an explicit `CdVaultItem` lookup table array.

```
[ Runtime Invocation ] 
         |
         v
[ Fetch Profile Details ] ----> Detect Pointers: "#cdVault['token']"
         |
         v
[ CdCliVaultController.resolveVaultReferencesInObject() ]
         |
         +---> Loop through keys recursively
         |
         +---> Match pointer key to cdVault array entry
         |
         +---> Is entry encrypted?
                  |
                  ├──> YES: Execute decryptValue() at runtime
                  └──> NO : Pull raw text value
         |
         v
[ Inject Plaintext Secret into Memory-Only Object ]

```

### 3.2 Core Architectural Subsystems

#### Controller 1: The Session Consumption Wrapper

This service handles active session loading, executing vault token processing before passing credentials down to target network layers.

```ts
export class SessionController {
  private logger = {
    logDebug: (msg: string, ctx?: any) => console.log(`[DEBUG] ${msg}`, ctx),
    logError: (msg: string) => console.error(`[ERROR] ${msg}`)
  };

  async getSession(profileName: string): Promise<any | null> {
    try {
      const svCdCliProfile = new CdCliProfileService();
      const resultCliConfig = await svCdCliProfile.loadProfiles();

      if (!resultCliConfig.state || !resultCliConfig.data) return null;

      const profile = resultCliConfig.data.items.find(
        (item) => item.cdCliProfileName === profileName
      );

      if (!profile || !profile.cdCliProfileData) {
        throw new Error(`Profile "${profileName}" could not be located.`);
      }

      const session = profile.cdCliProfileData.details.session;
      if (!session) {
        throw new Error(`No session details declared in profile "${profileName}".`);
      }

      // Execute dynamic runtime translation of pointers to resolved values
      const resolved = await CdCliVaultController.resolveVaultReferencesInObject(
        profile.cdCliProfileData.details,
        profile.cdCliProfileData.cdVault
      );

      // Re-hydrate session token field safely in memory
      if (resolved.session && typeof resolved.session.cd_token !== "undefined") {
        session.cd_token = resolved.session.cd_token;
      }

      // Transform JWT if plaintext string context exists
      if (session && (typeof session.jwt === "string" || session.jwt === null)) {
        (session as any).jwtObj = session.jwt
          ? {
              jwtToken: session.jwt,
              checked: false,
              checkTime: null,
              authorized: false,
              ttl: session.ttl ?? null,
            }
          : null;
      }

      return session;
    } catch (e) {
      this.logger.logError(`Session extraction exception: ${(e as Error).message}`);
      return null;
    }
  }
}

```

#### Controller 2: The Core Vault Compilation Engine

```ts
export class CdCliVaultController {
  private static logger = {
    logWarn: (msg: string) => console.warn(`[WARN] ${msg}`)
  };

  /**
   * Deep traverses an object structure, locating vault parameters 
   * and injecting hot values directly in-flight.
   */
  static async resolveVaultReferencesInObject<T>(
    input: T,
    cdVault: any[] = []
  ): Promise<T> {
    const isVaultRef = (val: unknown): val is string =>
      typeof val === 'string' && /#cdVault\['(.+?)'\]/.test(val);

    const recursiveResolve = async (obj: any): Promise<any> => {
      if (Array.isArray(obj)) {
        return Promise.all(obj.map(recursiveResolve));
      }

      if (typeof obj === 'object' && obj !== null) {
        const result: any = {};
        for (const key of Object.keys(obj)) {
          result[key] = await recursiveResolve(obj[key]);
        }
        return result;
      }

      if (isVaultRef(obj)) {
        const key = obj.match(/#cdVault\['(.+?)'\]/)?.[1];
        const vaultEntry = cdVault.find((v) => v.name === key);
        
        if (vaultEntry) {
          return vaultEntry.isEncrypted 
            ? await this.decryptValue(vaultEntry) 
            : vaultEntry.value;
        } else {
          this.logger.logWarn(`Vault reference validation failure: key '${key}' missing.`);
          return obj;
        }
      }

      return obj;
    };

    return recursiveResolve(input);
  }

  private static async decryptValue(entry: any): Promise<string> {
    // Cryptographic utility integration (e.g., AES-256-CBC runtime block implementation)
    return "decrypted-runtime-value";
  }
}
```

## 6. Guidelines for Core CorpDesk Developers

1. **Never hardcode raw production credentials** inside the `details` schema block. Always utilize the `#cdVault['<KEY_NAME>']` reference approach.
2. Ensure new architectural subsystems consistently invoke `CdCliVaultController.resolveVaultReferencesInObject()` before reading parameters from `details`.
3. If extending profile interfaces for new modules (e.g., database configs, messaging systems), append valid identifier entries directly to the `CdCliProfileTypes` execution tracking enum.
